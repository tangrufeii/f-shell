# FShell 项目追踪

最后更新：2026-04-06

## 项目目标

做一个基于 Rust + Tauri 2 的现代 SSH/SFTP 桌面工具，方向是：

- 真 SSH / 真 SFTP，不做假壳子
- 干净、克制、现代的桌面 UI
- 文件树、预览、在线编辑、上传、拖拽、剪贴板都要真能用
- 终端和文件操作体验要接近成熟桌面工具，而不是拼凑 demo

## 已完成

### 基础架构

- 已迁移到 Tauri 2
- 前端为 React + TypeScript + Vite
- 后端为 Rust + `ssh2`
- 已建立真实 SSH Shell 和真实 SFTP 能力

### 连接与终端

- 已支持真实 SSH 连接、断开、终端尺寸同步
- 已修好终端输入、焦点、光标漂移和回显问题
- 已支持终端命令历史、本地持久化
- 已支持 `Tab` 补全触发
- 已支持终端查找、清屏、历史命令回填

### 文件系统

- 已支持远端文件树懒加载
- 已支持目录展开/折叠
- 已支持图片预览
- 已支持文本文件预览与在线保存
- 已支持拖拽上传
- 已支持浏览器剪贴板图片/文件上传
- 已支持 Windows 原生剪贴板文件上传
- 已支持文件 / 目录下载
- 已统一远端路径为 POSIX `/` 形式，修复 Windows 客户端下目录项偶发混入反斜杠的问题

### 权限与状态

- 已支持根据远端 `uid/gid + mode` 推导基础读写进入权限
- 已支持文件树中的只读/无权限置灰
- 已支持不可读文件阻止预览
- 已支持不可进入目录阻止展开
- 已支持只读文件禁用保存
- 已修复连接失败后旧文件树残留的问题
- 已补齐连接表单的字段校验和错误反馈

### 交互与界面

- 已把连接配置改成弹窗，而不是长期占用布局
- 已将左侧主区域收敛到文件树
- 已把顶部栏整理为主机状态 + 常用操作
- 已做底部终端工具栏
- 已移除预览态下不该出现的终端工具栏
- 已禁用连接弹窗点空白误关闭
- 已为文件树接入右键菜单
- 已修正“在终端进入这里”的命令拼接，避免路径切换被多余参数干扰
- 已补齐文件树管理动作：新建文件、新建目录、重命名、删除
- 已修复自定义窗口按钮权限，支持拖动窗口与边缘缩放
- 已补产品级应用图标母版与 Windows 图标资产，替换占位图标
- 已接入 Tauri 2 在线更新检查 / 安装链路
- 已启用 updater 签名产物生成
- 已把 Windows NSIS / MSI 打包语言切到中文
- 已修复 Windows 发布版额外弹出控制台窗口的问题
- 已把 updater 更新源改成 GitHub Releases `latest.json`
- 已新增 GitHub Actions Windows 发版工作流

## 当前已知问题 / 待打磨

- 终端历史菜单虽然可用，但还不够像成熟工具
- 预览区仍缺少更强的格式化预览能力
- 实时主机指标（CPU、内存、负载）仍未接入
- 更新中心虽然已经有进度和偏好，但还缺少下载速度、剩余时间、失败原因分类等更细颗粒度状态
- GitHub Actions Release 仍需继续核实 `v0.2.8` 的最终发布结果与 `latest.json` 切换状态

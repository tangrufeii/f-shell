# FShell 用户体验优化待办

最后更新：2026-04-06

这个文件只记录两类事情：

- 明显提升用户体感的功能点
- 能降低困惑、减少误操作、提升完成效率的优化项

不在这里记纯技术重构，也不记“以后可能有用”的空想需求。

## 阶段进度

- 已完成一阶段：
  - 文件树权限状态可视化升级
  - 只读预览提示条
  - 只读文件的保存 / 粘贴禁用和编辑器只读保护

## P0：最该先做，做完就能明显变顺手

### 1. 多会话 / 多标签

- 支持同时打开多个 SSH 会话，而不是单窗口单会话
- 每个会话独立保留：
  - 当前路径
  - 文件树展开状态
  - 终端历史
  - 预览文件
- 标签上直接显示：
  - 会话名称
  - 主机
  - 连接状态
  - 未保存标记
- 支持快速切换、关闭其他、重连当前标签

用户价值：
现在单会话模型太憋屈，用户一旦要同时看两台机器、两套目录、两种环境，体验立马掉到 demo 水平。

### 2. 文件树权限可视化升级

- 区分三种状态：
  - 不可读
  - 不可进入
  - 只读
- 用更明确的视觉语言表达：
  - 图标
  - 文案
  - hover 详情
- 预览头部增加权限提示条
- 对不可操作动作给出禁用原因，而不是只让按钮灰掉

用户价值：
现在权限能力有了，但用户看不直观，容易陷入“为什么点不了”的弱智时刻。

### 3. 连接错误文案人话化

- 按错误类型拆开提示：
  - DNS 解析失败
  - 端口拒绝
  - 认证失败
  - 连接超时
  - 目标主机断开
- 每种错误给明确建议动作：
  - 检查地址
  - 检查端口
  - 检查密码
  - 检查代理 / VPN
  - 稍后重试
- 保留原始错误详情，但放到“展开详情”里

用户价值：
桌面工具不该把底层异常原样甩脸上，得告诉用户现在该干嘛。

### 4. 更新链路闭环自检

- 在旧版本客户端里完整走一遍：
  - 检查更新
  - 下载
  - 安装
  - 重启
  - 版本切换
- 对失败场景补提示：
  - GitHub 不可达
  - 签名不匹配
  - 下载中断
  - 安装器启动失败
- 在更新面板显示：
  - 下载速度
  - 剩余时间
  - 失败原因分类

用户价值：
在线更新不是“能弹出来”就算完，真要敢放出去，闭环必须跑实。

## P1：高收益打磨，能把“能用”推到“好用”

### 5. 命令历史增强

- 支持命令收藏
- 支持按关键字搜索历史
- 支持按目录、时间、收藏状态筛选
- 支持重复执行前先预览命令
- 支持最近执行结果状态标记

用户价值：
现在的历史命令已经比没有强，但还不够像正式工具，用户回用命令的效率还差一截。

### 6. 快速连接体验升级

- 顶部“已保存连接”支持：
  - 搜索
  - 置顶
  - 最近使用
  - 一键重连
- 支持“最近连接”区域
- 支持根据当前表单自动提示匹配的已保存配置
- 支持显示最后连接是否成功

用户价值：
连接是高频入口，入口效率低，整款工具的第一印象就垮。

### 7. 文件预览体系升级

- 补齐 PDF 预览
- Markdown 增加格式化预览 / 源码切换
- 代码文件增加更稳的语言识别
- 大文件预览前先提示大小和风险
- 二进制文件给更明确的说明，而不是只说“不能预览”

用户价值：
预览是桌面 SSH/SFTP 工具的核心能力之一，信息表达太弱会拖整机体验后腿。

### 8. 远端操作反馈更完整

- 上传、下载、删除、重命名、新建操作统一反馈样式
- 增加处理中、成功、失败三段状态
- 对长耗时任务显示进度或阶段
- 失败后支持一键重试

用户价值：
用户最怕“到底有没有在干活”。异步操作没有明确反馈，心态直接炸。

## P2：中期增强，能拉开和普通工具的差距

### 9. 双栏文件管理模式

- 支持“文件树 + 当前目录列表”双栏视图
- 支持更快的批量选择
- 支持更直观的排序：
  - 名称
  - 时间
  - 大小
  - 类型

用户价值：
现在偏树结构，适合导航，不够适合高频文件操作。

### 10. 主机状态信息面板

- 展示 CPU、内存、磁盘、负载
- 展示系统版本、内核、运行时长
- 展示网络延迟趋势
- 连接异常时给出状态异常提示

用户价值：
连接上机器后，用户第一件事往往是判断这台机器当前状态怎么样。

### 11. 会话恢复

- 应用重启后恢复上次打开的连接标签
- 恢复终端工作目录和预览文件
- 未保存内容恢复提示

用户价值：
桌面工具一关就失忆，体验太像网页 demo，不像生产力工具。

### 12. 终端增强能力

- 分屏
- 快捷命令面板
- 常用命令模板
- 发送中断信号
- 会话级环境信息展示

用户价值：
终端不是只要能输命令就完了，效率工具得把高频操作压到最短路径。

## P3：锦上添花，但必须在主链路稳定后做

### 13. 本地与远端传输任务中心

- 统一展示上传 / 下载队列
- 支持暂停、继续、取消
- 支持失败任务重试
- 支持查看最近传输记录

### 14. 更细的视觉状态系统

- 成功、警告、危险、处理中统一视觉规范
- 统一浮层、弹窗、下拉菜单的表面规则
- 收紧透明层叠，杜绝背景串透和层级污染

### 15. 新手引导和空态优化

- 首次启动引导
- 未连接时的操作提示
- 文件区空态、错误态、权限态文案统一

## 暂不建议现在做

- 插件系统
- 团队协作
- 云同步
- 花里胡哨的主题切换
- 大规模架构重写

原因：
主链路体验还没完全打磨到位，先别搞这些分散注意力的活。

## 建议执行顺序

1. 多会话 / 多标签
2. 文件树权限可视化升级
3. 连接错误文案人话化
4. 更新链路闭环自检
5. 命令历史增强
6. 快速连接体验升级
7. 文件预览体系升级
8. 远端操作反馈更完整

## 完成标准

每个优化项都至少回答这三个问题：

- 用户之前哪里难受
- 改完后具体顺了什么
- 有没有新增复杂度和维护成本

要是答不上来，就说明这需求八成还不值得做。
